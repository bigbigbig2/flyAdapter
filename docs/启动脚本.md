## 建图模式说明
这套启动后，系统是 **mapping 模式**，也就是建图模式。  
你前面用 RViz 配置打开时，报过 `map does not exist`，那说明当时 map 相关 TF 还没起来；而后面 `ros2 topic list` 已经能看到 `/map`、`/odom`、`/robot_pose`、`/slam/mode_status` 等导航话题。这就是建图/导航核心起来后的正常迹象。



# B. 加载已有地图进入定位/导航模式
这个模式我建议不要再用你原来那条：

```plain
sudo /opt/fftai/humanoidnav/scripts/run_real.sh \
  --map-dir /opt/fftai/nav/map \
  ...
```

因为你自己已经验证过，更稳的是：

1.  先正常起 `run_real.sh`
2.  再执行 `load_map.sh --map-path ...`

而且 `load_map.sh -h` 也明确写了它就是：  
**“Load a map and switch unified system to localization mode.”**

## 终端四：加载地图，切到 localization 模式
```bash
cd /opt/fftai/humanoidnav

source /opt/ros/humble/setup.bash
source /opt/fftai/humanoidnav/install/setup.bash

./scripts/load_map.sh \
  --map-path /opt/fftai/nav/map/ \
  --namespace GR301AB0025
```

你前面已经跑通过同类命令，系统返回过：

```bash
Successfully loaded map and switched to localization mode
```

这就是加载地图成功并切到定位模式的标志。



# 
**不用命名空间启动，反而对 RViz2 最省事**，因为所有 topic / TF 都回到根路径：

+ `/tf`
+ `/tf_static`
+ `/map`
+ `/odom`
+ `/scan`
+ `/plan`
+ `/robot_pose`

这样 HumanoidNav 自带的 `navigation_view.rviz` 大概率不用改 topic，也不会再因为 namespace 找不到 `map`。

但前提是：**HumanoidNav、adapter、load_map、RViz2 全部都不用 namespace，必须保持一致。**
