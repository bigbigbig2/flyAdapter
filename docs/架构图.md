

# 1. 这套导航 SDK，一句话怎么理解
HumanoidNav 这套系统，本质上是一个基于 **ROS2 Humble** 的完整导航栈。它把“传感器输入 → 建图/定位 → 地图管理 → Nav2 导航控制 → 监控与业务封装”串成了一条链。文档里明确写了它的核心模块包括 **SLAM、Nav2、Octomap、Perception、fourier_msgs**，并给了总架构图：IMU 和 LiDAR 输入，进入 3D 激光建图/定位，再生成 `/map` 和 `/optimize_map`，然后 Nav2 做规划和控制，最后输出速度。


https://support.fftai.com/docs/GR-X-Humanoid-Robot/GR3/SDK/Aurora-SDK/reference/aurora_dds_reference_CN/

# 2. 模块关系图
先给你一个“工程脑图版”：

```plain
[传感器层]
  ├─ IMU
  ├─ rslidar_points
  ├─ scan
  ├─ segmented_groundless_points
  └─ cloud_registered_gravity

        ↓

[建图 / 定位层]
  └─ lightning / run_unified_online
       ├─ IMU融合
       ├─ 激光惯性建图 / 定位
       ├─ 模式切换：mapping / localization
       ├─ 输出 /slam/mode_status
       ├─ 输出 /robot_pose
       ├─ 输出 /odom_status_code
       └─ 输出 /odom_status_score

        ↓

[地图层]
  ├─ map_server
  ├─ octomap_server
  ├─ /map
  └─ /optimize_map

        ↓

[Nav2 导航层]
  ├─ planner_server
  ├─ controller_server
  ├─ smoother_server
  ├─ bt_navigator
  ├─ waypoint_follower
  ├─ velocity_smoother
  └─ collision_monitor

        ↓

[桥接 / 业务封装层]
  ├─ raw_client_node
  └─ HTTP adapter (run_adapter.sh)

        ↓

[外部接口]
  ├─ ROS2 topic / service / action
  └─ HTTP /robot/... API
```

这个分层和官方文档里的系统概述是一致的：SLAM 负责 3D 点云建图、6DOF 位姿估计和 IMU 融合；Nav2 负责全局规划、局部控制和行为树；Octomap 负责 3D 到 2D 地图；Perception 负责点云预处理；`fourier_msgs` 负责监控和状态接口。

# 4. ROS2 里这套系统最关键的 4 个概念
## 4.1 Topic
Topic 是持续流动的数据。  
这套系统里最重要的 topic 可以分成几类：

### 传感器类
+ `/imu`
+ `/rslidar_points`
+ `/scan`
+ `/segmented_groundless_points`
+ `/cloud_registered_gravity`

### 定位/地图类
+ `/slam/mode_status`
+ `/robot_pose`
+ `/odom`
+ `/odom_status_code`
+ `/odom_status_score`
+ `/map`
+ `/optimize_map`

### 监控类
+ `/Humanoid_nav/health`
+ `/Humanoid_nav/events`

### 控制类
+ `/cmd_vel`
+ `/plan`
+ `/initialpose`
+ `/clear_map`

---

## 4.2 Service
Service 是“一次请求、一次响应”。  
这套系统最关键的 service 是：

+ `/slam/set_mode`
+ `/slam/load_map`
+ `/slam/save_map`
+ `/get_current_action`
+ `/cancel_current_action`

你前面已经确认过：

+ 服务类型应该用 `fourier_msgs/...`
+ `LoadMap` 字段是 `map_path`
+ `GetCurrentAction` 能成功回包，说明 service 通信层是通的。

---

## 4.3 Action
Action 用来做“长任务”，最核心的是：

+ `/navigate_to_pose`

它不是一次性返回，而是持续反馈导航进度，最后给成功/失败结果。文档里对 `NavigateToPose` 的 goal、result、feedback 都写了。`raw_client_node` 也是它的 action client。

---

所以所有接口都会自动带前缀，比如：

# 6. TF 到底是什么，为什么这么关键
这是你现在最需要建立的“概念模型”。

## 6.1 TF 的本质
TF 是“坐标系关系树”。  
导航系统不是只要有点云就能工作，它还必须知道：

+ 地图坐标系在哪
+ 机器人机体坐标系在哪
+ 雷达装在机器人哪个位置
+ odom 和地图怎么关联

这些都是 TF 负责描述的。

---

## 6.2 这套系统里最关键的 frame
通常是：

+ `map`
+ `odom`
+ `base_link`
+ `lidar_link`

### 典型关系
```plain
map -> odom -> base_link -> lidar_link
```

### 各段通常谁提供
+ `map -> odom`：定位系统
+ `odom -> base_link`：本体里程计 / 底盘状态
+ `base_link -> lidar_link`：静态 TF / 机器人模型

---

## 6.3 你现在为什么会卡死在这里
因为你现场已经确认了：

+ `/tf` 和 `/tf_static` 有 publisher 端点
+ 但 `tf2_echo odom base_link` 失败
+ `tf2_echo base_link lidar_link` 失败
+ `/odom` 也没有样本。

这说明当前系统不是“没有 TF 话题”，而是：

**你真正需要的 frame 关系没有建立出来，或者 frame 名和配置不一致。**

这也是为什么 unified 会不断报：

+ `Failed to get "lidar_link"->"base_link" frame transform`
+ `odom frame does not exist`
+ `init failed`。

# 8. 接口速查表（你最常用的）
## 模式切换
### `/slam/set_mode`
+ 类型：`fourier_msgs/srv/SetMode`
+ 参数：`mode`
+ 支持：`mapping / slam / 0`，`localization / loc / 1`。

---

## 模式状态
### `/slam/mode_status`
+ 类型：`std_msgs/msg/String`
+ 取值：`mapping` / `localization`。

---

## 地图加载
### `/slam/load_map`
+ 类型：`fourier_msgs/srv/LoadMap`
+ 参数：
    - `map_path`
    - `x`
    - `y`
    - `z`
    - `yaw`。

---

## 初始位姿
### `/initialpose`
+ 类型：`geometry_msgs/msg/PoseWithCovarianceStamped`
+ 用于给 localization 设初始位置。

---

## 机器人位姿
### `/robot_pose`
+ 类型：`geometry_msgs/msg/PoseStamped`
+ 含义：机器人在 `map` 坐标系下的 6DOF 位姿。

---

## 里程计
### `/odom`
+ 类型：`nav_msgs/msg/Odometry`
+ 含义：机器人里程计数据。

---

## 定位状态
### `/odom_status_code`
+ 类型：`std_msgs/msg/Int8`
+ 含义：
    - `0 = IDLE`
    - `1 = INITIALIZING`
    - `2 = GOOD`
    - `3 = FOLLOWING_DR`
    - `4 = FAIL`。

### `/odom_status_score`
+ 类型：`std_msgs/msg/Int8`
+ 含义：定位置信度评分。

---

## 单点导航
### `/navigate_to_pose`
+ 类型：`nav2_msgs/action/NavigateToPose`
+ 用途：导航到指定目标点。

---

## 当前动作
### `/get_current_action`
+ 类型：`fourier_msgs/srv/GetCurrentAction`
+ 用途：查询当前导航动作状态。  
你现场已经验证过它能成功回包。
