```plain
机器人 enp2s0 IP: 192.168.137.220

 |
路由器 / PC 网关 192.168.137.2

```

+ 局域网通信没问题，可以 SSH、DDS 真机测试 
+  外网访问不通，因此 `sudo apt update`、pip install 外网包会失败



#### 方案 1：路由器 NAT / 共享外网
+  确认网关 192.168.137.2 是实际能访问互联网的设备（PC 或路由器） 
+  如果是 PC，需要启用 **Internet Connection Sharing** 或桥接网络 
+  如果是路由器，需要确保 LAN 口能访问 WAN（外网） 

#### 方案 2：直接把机器人接到有外网的路由器
+  插入一个有互联网的路由器 LAN 口 
+  配置机器人获得 DHCP IP 或手动配置静态 IP 
+  确认默认网关是路由器 LAN IP 
+  再执行 ping 8.8.8.8 测试外网 

#### 方案 3：离线安装
+  如果外网短期无法接入，可以在本机用 `pip download` 或官方 SDK `.zip/.tar` 离线安装 
+  适合只做 Aurora 真机测试，不依赖外网更新包



这里采用方案二：

## 方案 2：机器人通过 Linux PC 上外网
### 原理
+  Linux PC 有两张网卡： 
    - `eth0` 连接外网（上网接口） 
    - `eth1` 连接机器人（局域网接口） 
+  PC 开启 IP 转发 + 配置 NAT，让机器人通过 PC 上网 
+  机器人设置默认网关为 PC 的局域网 IP 

---

### 步骤
#### 1️⃣ 在 Linux PC 上检查网卡
```plain
ip addr
```

示例：

```plain
eth0: 192.168.1.100  # 外网接口
eth1: 192.168.137.1  # 机器人连接
```

---

#### 2️⃣ 开启 IP 转发
```plain
sudo sysctl -w net.ipv4.ip_forward=1
# 永久开启可以编辑 /etc/sysctl.conf 添加：
# net.ipv4.ip_forward=1
```

---

#### 3️⃣ 配置 NAT 和转发规则
```plain
# 假设 PC 上网接口 eth0，机器人连接接口 eth1
sudo iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
sudo iptables -A FORWARD -i eth1 -o eth0 -j ACCEPT
sudo iptables -A FORWARD -i eth0 -o eth1 -m state --state ESTABLISHED,RELATED -j ACCEPT
```

+  这三条规则让机器人通过 PC 上网 
+  建议保存 iptables 规则，防止重启丢失 

---

#### 4️⃣ 在机器人上设置默认网关
假设 PC 局域网 IP 是 `192.168.137.1`：

```plain
sudo ip route add default via 192.168.137.1 dev enp2s0
```

+  检查路由： 

```plain
ip route
```

+  测试外网： 



```plain
ping 8.8.8.8 -c 4
ping mirrors.aliyun.com -c 4
```

---

### 注意事项
1.  确保 PC 的防火墙允许转发和 NAT 
2.  如果机器人要长期上网，建议在 PC 上把 iptables 规则保存并开机自动加载 
3.  这种方法只在机器人通过网线连接到 PC 的情况下使用 
4.  一旦 PC 上网断开或重启，机器人可能需要重新配置默认网关 

---

💡 **总结**

+  原理：PC 当路由器，机器人通过它访问外网 
+  步骤：开启 IP 转发 + 配 NAT + 配机器人默认网关 
+  优点：无需在实验室改动路由器，适合短期或局域网测试
